# 超勤簿入力アプリ

医師の超過勤務命令簿を簡単に入力し、公式Excel様式・PDF・印刷で出力できるWebアプリ。

## ファイル構成

| ファイル | 役割 |
|---|---|
| `index.html` | アプリ本体 |
| `choukinbo_template.xlsm` | 公式Excelテンプレート（**index.htmlと同じ場所に必須**） |

この2ファイルは必ずセットで、同じフォルダ／同じ階層に置いてください。

## 主な機能

- 報告期間（前月21日〜当月20日）の全日をカード型UIで一覧入力
- クイック時刻ボタン、業務テンプレート、朝カンファ等の定型入力
- 平日/休日 × 5-22時/22-5時 を自動集計
- ブラウザに自動保存（LocalStorage）、JSONバックアップ可
- **公式Excel書き出し** … 元のExcel様式そのままで出力（後述）
- PDF出力 → iOSの共有シートからメール/LINE/AirDropで送信可
- A4縦1ページの命令簿フォーマット印刷

## 公式Excel書き出しについて

「📗 公式Excel」ボタンは、公式テンプレート `choukinbo_template.xlsm` をそのまま開き、
「命令簿」シートの入力セルだけを書き換えて出力します。ゼロから作り直すのではなく
公式ファイルに値を流し込む方式のため、次のものはすべて完全に保持されます。

- セルの書式・罫線・フォント・結合セル
- すべての数式（④勤務の区分・小計・支給率の自動計算）
- VBAマクロ、条件付き書式、印刷範囲

出力ファイル名は `超勤簿_氏名_YYYY年M月.xlsm`。開いた瞬間に自動再計算されるよう
設定済みなので、Excelで開けば合計時間まで正しく表示されます。

※この機能には JSZip ライブラリ（CDN読込）を使うため、オンライン環境が必要です。

## GitHub Pages で公開する手順

### 1. リポジトリを作成
1. GitHub にログイン → 右上の「+」→ **New repository**
2. Repository name は何でも可（例: `chokin`）
3. **Public** を選択（Privateだと有料プランが必要）
4. **Create repository**

### 2. ファイルをアップロード
1. リポジトリのページで「**uploading an existing file**」リンクをクリック
2. `index.html` と `choukinbo_template.xlsm` の**両方**をドラッグ&ドロップ
3. 一番下の **Commit changes** をクリック

### 3. GitHub Pages を有効化
1. リポジトリ上部の **Settings** タブ
2. 左メニューの **Pages**
3. **Source** で「Deploy from a branch」を選択
4. **Branch** を `main` / `/ (root)` に設定して **Save**
5. 数分待つ（上部に "Your site is live at ..." と表示される）

### 4. アクセス
URL は `https://(あなたのユーザー名).github.io/(リポジトリ名)/`

例: ユーザー名が `taro` でリポジトリが `chokin` なら `https://taro.github.io/chokin/`

## iPhoneでアプリ化
1. SafariでアクセスURLを開く
2. 共有ボタン → **ホーム画面に追加**
3. ホーム画面のアイコンから起動するとアプリのような見た目になる

## データについて
- 入力データはブラウザの LocalStorage に保存される
- **デバイスごと・ブラウザごとに別管理**（iPhoneのSafariとPCのChromeは別）
- 複数デバイスで同じデータを使いたい場合は「メニュー → JSONエクスポート/インポート」で同期
- リポジトリを公開しても、入力データは GitHub には一切送信されない（完全にブラウザ内で動作）
